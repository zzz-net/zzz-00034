# 本地巡检日志分析看板

一款面向运维人员的本地桌面级数据分析工具，用于导入传感器数据、人工备注和告警信息，通过配置阈值自动识别异常事件并归并相关证据，支持事件复核流程和多格式导出。

## 功能特性

- 📊 **多源数据导入**：支持传感器 CSV、人工备注 CSV、告警 JSON 三种格式
- ⚙️ **灵活阈值配置**：温度上下限、电压上下限、离线时长、合并窗口均可自定义
- 🔍 **智能异常检测**：基于阈值自动识别温度异常、电压异常、离线异常
- 📦 **事件智能归并**：同设备相近时间内的多条证据自动归并为一个事件
- ✅ **事件复核流程**：支持待处理、已确认、误报、已关闭四种状态流转
- 📝 **处理记录**：记录处理人、备注、关闭时间等信息
- 💾 **本地持久化**：数据保存在浏览器 LocalStorage，重启后状态一致
- 📤 **多格式导出**：支持 CSV 和 JSON 格式导出事件及证据数据
- 🔄 **导入去重**：重复导入同一批次不会创建重复事件
- ⚠️ **错误报告**：缺少 device_id、时间无法解析等错误会报告具体行号和字段

## 完整使用流程

以下是一次完整的导入、分析、复核、关闭和导出流程：

### 1. 配置阈值

打开应用后，在左侧「阈值配置」面板中设置：

- **温度下限**：-10°C
- **温度上限**：60°C
- **电压下限**：200V
- **电压上限**：250V
- **离线时长阈值**：5 分钟
- **事件合并窗口**：30 分钟

点击「保存配置」按钮保存。

> 修改阈值后会自动重新分析所有已有数据

### 2. 导入样例数据

点击右上角「导入数据」按钮，依次导入以下三个文件（位于 `public/sample_data/` 目录）：

#### 2.1 导入传感器数据

1. 选择「传感器数据」标签
2. 上传 `public/sample_data/sensor_data.csv`
3. 查看导入结果：60 条记录成功导入（DEV-001:13条、DEV-002:19条、DEV-003:9条、DEV-004:19条）

#### 2.2 导入人工备注

1. 选择「人工备注」标签
2. 上传 `public/sample_data/manual_notes.csv`
3. 查看导入结果：12 条记录成功导入

#### 2.3 导入告警数据

1. 选择「告警数据」标签
2. 上传 `public/sample_data/alarm_data.json`
3. 查看导入结果：12 条记录成功导入

> 重复导入同一文件会被系统识别为重复批次并跳过

### 3. 查看事件列表

导入完成后，中间「事件列表」区域会显示自动检测出的异常事件（共 4 个事件）：

- **DEV-001**：温度异常 + 告警 + 备注，6 条证据，08:45~09:30（待处理）
- **DEV-002**：电压异常 + 离线 + 告警 + 备注，20 条证据，08:20~10:35（待处理）
- **DEV-004**：温度异常 + 告警 + 备注，2 条证据，09:30（待处理）
- **DEV-004**：温度异常 + 告警 + 备注，11 条证据，10:30~13:30（待处理）
- **DEV-003**：无异常（运行正常，仅有正常巡检备注，不触发事件）

可以按状态和设备筛选事件。

### 4. 事件复核

点击一个事件，在右侧「事件详情」面板中查看：

- 事件基本信息（设备、时间、持续时长）
- 证据时间线（传感器异常、人工备注、告警按时间排列）
- 处理人和备注

#### 4.1 确认异常

1. 输入处理人姓名（如「张工」）
2. 点击「确认异常」按钮
3. 事件状态变为「已确认」

#### 4.2 标记误报

1. 对于确认是误报的事件
2. 点击「标记误报」按钮
3. 事件状态变为「误报」

#### 4.3 关闭事件

1. 事件处理完成后
2. 点击「关闭事件」按钮
3. 事件状态变为「已关闭」，并记录关闭时间

### 5. 添加备注

1. 在事件详情的「备注」区域点击「编辑」
2. 输入备注内容
3. 点击「保存」

### 6. 导出数据

点击右上角「导出数据」按钮：

- **CSV 格式**：可选择导出全部、仅事件或仅证据
- **JSON 格式**：导出包含事件和证据的完整数据

导出文件会自动下载到本地。

### 7. 数据持久化验证

1. 确认事件状态、备注、处理人等信息
2. 刷新浏览器页面
3. 重新打开应用，验证数据是否与刷新前一致
4. 导出数据，验证导出内容与保存前一致

## 数据格式说明

### 传感器 CSV

必需字段：

| 字段 | 类型 | 说明 |
|------|------|------|
| device_id | string | 设备唯一标识 |
| timestamp | string | 时间戳，支持 ISO 格式、YYYY-MM-DD HH:mm:ss 等 |
| temperature | number | 温度（°C） |
| voltage | number | 电压（V） |
| is_online | boolean | 是否在线（true/false, 1/0, yes/no） |

示例：
```csv
device_id,timestamp,temperature,voltage,is_online
DEV-001,2024-01-15 08:00:00,25.5,220.5,true
```

### 人工备注 CSV

必需字段：

| 字段 | 类型 | 说明 |
|------|------|------|
| device_id | string | 设备唯一标识 |
| timestamp | string | 时间戳 |
| content | string | 备注内容 |
| author | string | 备注作者 |

示例：
```csv
device_id,timestamp,content,author
DEV-001,2024-01-15 08:45:00,设备温度异常上升,张工
```

### 告警 JSON

支持数组格式或包含 `data`/`alarms`/`items` 数组的对象。

每条告警必需字段：

| 字段 | 类型 | 说明 |
|------|------|------|
| device_id | string | 设备唯一标识 |
| timestamp | string | 时间戳 |
| alarm_type | string | 告警类型 |
| level | string | 告警级别（info/warning/critical 等） |
| description | string | 告警描述 |

示例：
```json
[
  {
    "device_id": "DEV-001",
    "timestamp": "2024-01-15 08:45:00",
    "alarm_type": "温度过高",
    "level": "warning",
    "description": "设备温度超过阈值"
  }
]
```

## 错误处理

系统会对导入数据进行校验，发现错误时：

- **缺少 device_id**：报告行号和字段名，跳过该行
- **时间无法解析**：报告行号和原始时间值，跳过该行
- **数值字段无效**：报告行号和字段名，跳过该行
- **阈值配置非法**：报告具体字段和错误信息，拒绝保存，保留旧配置

所有校验错误会在导入弹窗中统一展示，不中断导入流程，有效数据会正常导入。

## 技术栈

- **前端框架**：React 18 + TypeScript
- **构建工具**：Vite 6
- **样式方案**：Tailwind CSS 3
- **状态管理**：Zustand
- **图标库**：Lucide React
- **数据存储**：浏览器 LocalStorage

## 注意事项

- 数据仅保存在浏览器本地，清除浏览器数据会导致数据丢失
- 建议定期导出数据备份
- 推荐使用 Chrome、Edge、Firefox 等现代浏览器
- 单设备数据量较大时可能会有性能影响
